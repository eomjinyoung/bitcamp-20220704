외부라이브러리 사용 - CDN(Content Delivery Network) 방식

- 외부라이브러리를 사용할 때 원래의 URL을 그대로 링크 건다.

[장점] 
- 우리의 서버에 다운로드 할 필요가 없기 때문에 관리가 편하다.
- 웹 브라우저가 각 사이트 별로 파일을 캐시에 담아 두기 때문에 
  사용자가 다른 시스템을 방문했을 때 이미 다운로드한 라이브러리라면  
  다시 받지 않기 때문에 페이지 로딩 속도가 빨라진다. 

[단점]
- 라이브러리가 있는 서버에 연결되지 않으면 우리 시스템의 기능을 온전히 사용할 수 없다.
- 라이브러리가 외부에 있기 때문에 우리 마음대로 통제할 수 없다.
  라이브러리가 있는 서버쪽에서 해킹 코드를 삽입하더라도 막을 방법이 없다.
